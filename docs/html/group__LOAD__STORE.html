◆ __ld_Rx_mem_br_replace_h

#define __ld_Rx_mem_br_replace_h (   __n,
  __addr1,
  __addr2

Value:
do { \
__ld_vec_asm_br(__addr1, __addr2); \

} while(0)
Prototype:
void __ld_Rx_mem_br_replace_h(const unsigned int n, Type *addr1, Type *addr2)
Description:
Load a word with bit reversal into an R register
Load a word from DMEM into the most significant word of Rn, where n is an
integer from 0 to 7. The address used for the memory access is computed by
applying a bit reversal re-order algorithm on the address contained in the
internal bit reversal address register. The address contained in the internal
bit reversal address register is updated to addr1 and addr1 is post-incremented
by addr2.
This intrinsic implements the combination of intrinsic __ld_vec_asm_br and
__ld_Rx, both defined in this document.
The bit reversal reorder algorithm must have been previously initialized with 
__set_br. 
The internal bit reversal address used in the access is the one set by the 
last bit reversal instruction (__set_br, load or store with bit reversal).

Example of using a sequence of loads or stores with bit reversal:

(1) __set_br(addr1, fft_size); // Sets the internal bit reversal address to
                               // addr1 and the bit reversal mode according to
                               // fft_size.
(2) addr1 += addr2;            // Updates addr1 so that the next load or store
                               // with bit reversal will use addr1 + addr2 to
                               // update the internal bit reversal address.
(3) __ld_Rx_mem_br_replace_h(0, addr1, addr2); // Uses the internal bit
                              // reversal address, set by (1), the fft_size 
                              // set by (1), assigns the current value of 
                              // addr1 (written by (2)) to the internal bit
                              // reversal address. 
                              // Adds addr2 to addr1. 
(4) __ld_Rx_mem_br_replace_h(0, addr1, addr2); // Uses the internal bit 
                              // reversal address, set by (3), and the 
                              // fft_size set by (1), assigns the current
                              // value of addr1 (written by (3)) to the 
                              // internal bit reversal address.
                              // Adds addr2 to addr1. 
Type can be any type in the Scalar data types section defined in 2.1.
Underlying assembly:
ld.br [aX]+aY ld.replace_h Rn
Support:
VSPA

◆ __st_vec_asm

#define __st_vec_asm (   __addr)    __asm volatile ("st [%0]+0": "=mp"(__addr) : );
Prototype:
void __st_vec_asm(Type *addr)
Description:
Store a vector to the aligned instance of an address
Store a vector from R[rSt] to DMEM at the aligned address obtained by aligning
'addr' to a line boundary (ignoring the least significant log2(__AU_COUNT__ * 4)
bits of 'addr'). Even though it accepts an unaligned address as an argument,
this intrinsic does NOT perform an unaligned vector store, as there is no support
for unaligned vector stores. Instead, the intrinsic is meant to be used in 
conjunction with unaligned vector load intrinsics, such as __ld_vec_asm, which
may use scalar data pointers.

Note:
Use with caution, prefer __st_vec whenever possible.
Underlying assembly:
st [aX]+0
Support:
VSPA

Prototype:
void __st_vec_asm_llr(EnumMode mode, Type *addr1, Type *addr2)
Description:
Store a vector to an unaligned address using bit likelihood ratio (LLR).
st.llr4 [aX]+aY reads a vector from R[rSt], compresses each 16-bit half-fixed
value to a nibble (4-bit wide), and then writes to one quarter of a DMEM line
at addr1. The addr1 address must be quarter-line aligned.
addr1 is post-incremented by addr2.
st.llr4half [aX]+aY reads a vector from R[rSt], compresses each 16-bit half-float
value to a nibble (4-bit wide), and then writes to one quarter of a DMEM line
at addr1. The addr1 address must be quarter-line aligned.
addr1 is post-incremented by addr2.
st.llr8 [aX]+aY reads a VRA register, compresses each 16-bit half-fixed value
to an octet (8-bit wide), and then writes to one half of DMEM line. The addr1
address must be half-line aligned.
addr1 is post-incremented by addr2.
st.llr8half [aX]+aY  reads a VRA register, compresses each 16-bit half-float
value to an octet (8-bit wide), and then writes to one half of DMEM line. The 
addr1 address must be half-line aligned.
addr1 is post-incremented by addr2.
Underlying assembly:
st.llr4 [aX]+aY st.llr4half [aX]+aY st.llr8 [aX]+aY st.llr8half [aX]+aY
Support:
VSPA
