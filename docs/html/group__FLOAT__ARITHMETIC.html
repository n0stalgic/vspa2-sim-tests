◆ __padd

#define __padd ( )    __asm("padd");
Prototype:
void __padd()
Description:
Pre-add S1 and S2
Adds 2 vectors of half-fixed numbers as loaded from the VRA into S2 and S1.
This instruction is relevant only in "padd" AU precision. Under "padd" mode
of AU precision, no type conversion happens on the source register load and
the inputs are always treated as half-fixed numbers.
The operation performed is :

Real/Complex mode:
for (i=0; i<256; i+=1)
   sauOut[i] = S2[i] + S1[i]

The output vector from the pre-adder is available at the SAU output either for
use by any of special arithmetic unit (sau) instructions or for writing to VRA.
If written back to VRA, only half of the output vector containing every
sample is written back.
Underlying assembly:
padd
Support:
VSPA2
